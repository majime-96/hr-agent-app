# システム原型ナレッジ（因果ループ図の類推用）

出典：ピーター・センゲ / ダニエル・キム、「システム・シンキング トレーニング ブック — 持続的成長を可能にする組織変革のための8つの問題解決思考法」（宮川雅明・石坂正 訳、日本能率協会マネジメントセンター、2002）

---

## このファイルの使い方（AIへの注記）

- これは、因果ループ図を描くときに、**構造を見極めるための「ものさし」**として参照する資料です。
- **名称（「問題の転嫁」など）は、ユーザーには絶対に出さないでください。** あくまで内部での見立てに使い、ユーザーに見せる図や説明では「平易な言葉の因果ループ」として描きます。
- **型にそのまま当てはめないでください。** どの型にもぴったり来なければ、無理に当てはめず、素の因果ループとして描いて構いません。型は「引き出し」であって、「答え」ではありません。
- 各型の「見立ての問い」に複数当てはまる場合は、一つに絞らず、展開した構造を図にしてください。

## 8つのシステム原型

### 1. 応急処置の失敗
- **定義：** 問題に応急処置的な手を打つと、一時的に症状は和らぐが、やがて問題が再発し、同じことを繰り返すうち、根本解決への意欲が下がる。応急処置を重ねるほど、問題は深刻になっていく。
- **因果構造：** 問題の症状 → 応急処置 →（一時的に）症状が緩和 → それを通じて意図しない悪い結果が出る → 症状がさらに悪化 → また応急処置。
- **見立ての問い：** その場しのぎの対処をすることで、かえって事態を悪くしていないか？
- **ソーシャル企業のひとり人事での例：** 退職者が出るたびに急いで欠員補充の採用をするが、一時的に人数は戻るが定着しない人が増え、教育の負荷が現場にまたかかり、さらに退職が増えていく。

### 2. 問題の転嫁
- **定義：** 根本原因ではなく、対症療法で問題をそのままにし、一時的に楽になるが、根本解決への意欲が下がり、また問題が再発。また対症療法に頼るという繰り返し。根本に向き合う力がただただ衰える。
- **因果構造：** 問題 →（対症療法）→ 一時的に軽減 → 根本策に取り組む意欲が下がる → 根本原因は残る → 問題が再発 → また対症療法。
- **見立ての問い：** 根本的な手当てを避けて、最近の対処を繰り返していないか？
- **ソーシャル企業のひとり人事での例：** 評価の不満が出るたびに個別に説明してなだめるが、その場は収まる。評価基準そのものを整える作業は手がつかないままで、また次の不満が出る。

### 3. 成長の限界
- **定義：** 当初はうまくいっていたが、ある時点でシステムの限界に遭遇し、それ以上良い結果が出なくなる。成長を妨げる要因がはびこり、成長を阻害する制約に変わる。
- **因果構造：** 努力 → 成果（拡張ループ）→ 何らかの制約に突き当たる →（バランスループ）成果が頭打ちになる。
- **見立ての問い：** これまで順調だったのに、最近、頭打ちになっていないか？ 成長を止めている制約は何か？
- **ソーシャル企業のひとり人事での例：** 創業期は少人数の熱意で急成長したが、30名を超えたあたりから制度や役割の未整備が制約になり、それ以上人が増えても動かなくなっている。

### 4. 目標のなし崩し
- **定義：** 本来めざす目標が、現実とのギャップに疲れ、いつしか崩れ気味に下げられて、そのギャップを「減らす努力」ではなく、目標の「ほうを下げて」帳尻を合わせてしまう。
- **因果構造：** 目標と現実のギャップ →（本来は）現実を引き上げる努力 → だが疲れるので目標を下げる → ギャップは一時的に消える → 達成感を下げ続ける。
- **見立ての問い：** うまくいっていたことが、当初の基準や理想を下げていないか？
- **ソーシャル企業のひとり人事での例：** 「全員と月1回1on1をしたい」と決めたが、忙しくてできない人だけやる、とそこから基準を下げていく。いつのまにか形骸化している。

### 5. 成長と投資不足
- **定義：** 投資を怠ると生産・供給が拡大できない。投資効果が出るまでに遅れがあると、需要に応じられず、評判・需要が下がる。成長が止まる。
- **因果構造：** 需要の増大 → 供給能力が足りない → 投資すべきだが抑制する →（遅れ）能力が上がらない → 需要に応じられず評判・需要が下がる → 成長が止まる。
- **見立ての問い：** 先を見据えた投資（人・仕組みへの投資）を見送ってしまい、機会を逃していないか？
- **ソーシャル企業のひとり人事での例：** 事業は伸びているのに、採用や育成への投資をまだ早いと先送りにし、人が必要なときに育っていない。気づいたらその事業機会を逃している。

### 6. 成功には成功が
- **定義：** 一方が成功すると資源がそちらに集まり、成功がさらに続く。もう一方は資源を奪われ、衰退する。成功の選択が最適でなくても、それ自体が自己強化される。
- **因果構造：** AとBが同じ資源を奪い合う → Aが少し先行 → 資源がAに集まる → Aがさらに成功、Bが衰退 → 結果が固定化。
- **見立ての問い：** 特定の人・部署・事業にぐっと資源（人・予算・関心）が集まり、他がみるみる細っていないか？
- **ソーシャル企業のひとり人事での例：** 成果を出すエース部門に良い人材と予算が集中し、立ち上げ期の新規事業に人が回らない。新規事業はますます育たず、エース依存が強まる。

### 7. エスカレート
- **定義：** ライバル関係にある二者が、相手に対抗して行動を強め合い、際限なく激化していく。どちらも相手が「やめてくれ」と止められない。
- **因果構造：** Aの行動 → Bが脅威を感じ対抗 → Bの行動 → Aが脅威を感じ対抗 → 双方がエスカレート。
- **見立ての問い：** 対立する二者が、互いに張り合って、消耗し合っていないか？
- **ソーシャル企業のひとり人事での例：** 2つの部署がお互い「もっと増員してくれ」という要求を強め合い、限られた人員を奪い合って、組織全体が疲弊している。

### 8. 共有地の悲劇
- **定義：** 共有資源を各自が自己都合・自己利益のために使うと、資源に過大な負荷がかかり、最終的に全員の利益が減り、資源が枯渇すればシステムそのものが崩壊しかねない。
- **因果構造：** 各自が共有資源を利用 → 各自の利益は増える → 全体の資源が消耗 →（遅れ）資源が枯渇 → 全員の利益が減る。
- **見立ての問い：** みんなが頼る共有の資源（特定の人・予算・時間）が、使われすぎて枯渇されていないか？
- **ソーシャル企業のひとり人事での例：** 「困ったときの頼れるベテランのその人」に全部署が頼りすぎ、その人の時間という共有資源が枯渇。その人が疲弊・離職すると、組織全体が一気に動かなくなる。

## 類推の手順（内部処理）

1. 集まったできごと・パターンから、上の8つの「見立ての問い」に照らす。
2. 当てはまりそうな型を1〜2つ、内部で見立てる（一つに絞らなくてもいい）。
3. その型の「因果構造」を参考に、変数とつながりを組み立てる。
4. ただし、**ユーザーに見せるのは型名ではなく、平易な言葉の因果ループ**。型はあくまで構造を見極めるための一刷り。
5. どの型にもぴったり来なければ、型を捨てて、素の因果ループとして描く。
